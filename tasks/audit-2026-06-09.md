# Audit: Caloura @ main — 2026-06-09

Stack/subpackages: single Swift/macOS app (~34.6k LOC, 281 files)
Dimensions: power efficiency · correctness/crash-vectors · product polish/UX · stealth preservation · dead code/DRY/quality
Coverage: full (5 parallel read-only auditors + orchestrator red-team verification pass)

Red-team corrections applied to raw auditor output:
- OSLog dynamic interpolations are `.private` by default — the "logs broadcast capture details publicly" findings were demoted; only static message text (capture *timing*) leaks.
- The 50 ms cursor reprime loop is gated by `cursorActive` (session-scoped, not always-on) — demoted critical → high.
- All `file:line` claims for Critical/High findings were re-verified against source by the orchestrator.

## Critical / High (fix before ship)

- **[critical][stealth][reachable]** entire codebase — `sharingType` appears NOWHERE (verified via grep). `NSWindow.sharingType` defaults to `.readOnly`, so every Caloura window is visible to screen recordings and Zoom/Teams shares: pinned screenshot panels (float over all Spaces — `PinnedScreenshotWindow.swift:80–89`), Quick Access chip (`QuickAccessOverlay.swift:24`), countdown (`CountdownOverlay.swift:57`), capture overlays (`CaptureOverlayWindow.swift:33`, `ScreenSelectionOverlayWindow.swift:8`), and every `SingleWindowPresenter` window (Preferences, Onboarding, History, Annotation, Beautify, Redaction, tips — `SingleWindowPresenter.swift:52`). Fix: set `sharingType = .none` in a shared post-init helper applied to every window/panel; verify behavior on the target OS (`.readOnly` is deprecated on recent macOS, `.none` remains honored). SCK self-capture already excludes Caloura via `excludedApplications` (`ScreenCaptureManager+SCKCapture.swift:179`), so this does not affect the app's own captures.

- **[high][stealth][reachable]** `SingleWindowPresenter.swift:38,71,94` — unconditional `NSApplication.shared.activate()` on every window show steals focus from the frontmost app and pulls the accessory app into Cmd-Tab. Contextual tips and the nag dialog can fire this without user intent. Fix: drop the blanket calls; activate only when the user explicitly opened the window (pass a flag), otherwise rely on `makeKeyAndOrderFront`.

- **[high][power][reachable]** `CaptureCursorController.swift:106,191–204` — `scheduleMaintenanceReprime()` self-reschedules every 50 ms (20 Hz MainActor wakeups) for the entire duration of every area/fullscreen capture session. CAUTION: this is one layer of the documented cursor-race defense (CLAUDE.md / lessons.md) — do not just delete it. Fix: widen the interval substantially (e.g. 250–500 ms) or replace with event-driven reprime (didBecomeActive + activeSpaceDidChange + one-shot post-start), then re-verify the original cursor race scenario.

- **[high][power][reachable]** `ScreenCaptureManager.swift:263–276` — persistent `didBecomeActiveNotification` observer prewarms `SCShareableContent.current` on every app activation once the 60 s TTL lapses, even when no window capture is imminent. Fix: gate prewarm on recent window-capture use, or drop passive prewarm and warm lazily on demand.

- **[high][power][reachable]** `OnboardingView.swift:182–199` + `PermissionCoordinator.swift:356` — nested polling during permission repair: outer 10×3 s loop, inner 350 ms loop with up to 5 SCK screenshot probes per tick → up to ~50 `SCScreenshotManager.captureImage` calls in 30 s while the user is in System Settings. Fix: one validation per outer tick; let `didBecomeActive` drive revalidation instead of fixed timers.

- **[high][polish][reachable]** `UpdateManager.swift:57–63` + `MenuBarView.swift:73–77`, plus `PinnedScreenshotWindow.swift:63`, `HistoryView.swift:252`, `BeautifyPreviewOverlay.swift:129`, `CaptureDistributionService.swift:170,192` — raw `error.localizedDescription` shown verbatim in the menu bar and four other surfaces ("An SSL error has occurred…"). Fix: map known Sparkle/network/clipboard error categories to friendly strings; generic fallback otherwise.

- **[high][polish][reachable]** `HistoryView.swift:123–140` — "Open" / "Show in Finder" / "Copy" context-menu items silently no-op for clipboard-only captures (`filePath` empty, guard-return with no feedback). Fix: disable or hide items when `item.filePath.isEmpty`.

- **[high][polish+correctness][reachable]** `BeautifyPreviewOverlay.swift:90–116` — two compounding issues: (a) Copy/Save buttons silently no-op while `previewImage == nil`; (b) `generatePreview()` sets `isProcessing = true` with no `defer` — `.task(id:)` cancellation on theme switch leaks the flag and the spinner sticks forever (violates the project's defer-clear rule). Fix: `.disabled(previewImage == nil || isProcessing)` + `defer { isProcessing = false }`.

- **[high][polish][reachable]** `PreferencesView.swift:280–287` vs `DiagnosticsReporter.swift:36` — "Share anonymous diagnostics" copy claims the toggle is inert ("stores your preference for a future release"), but it actively gates local MetricKit payload capture today. Misleading copy on a privacy-sensitive toggle. Fix: rewrite to describe actual behavior (local crash/hang diagnostics, no uploads).

- **[high][quality]** `UpdateManager.swift:1,79,90,125–130` + `UpdateManager+Sparkle.swift` — sole Combine usage in production (`canCheckForUpdatesPublisher` + `cancellables` sink) violates the codebase standard (@Observable, async/await, no Combine) and forces the protocol + test mocks to import Combine. Fix: replace with `withObservationTracking` loop (pattern already used in `observeAutomaticUpdatesSetting`), delete the publisher requirement.

## Clean dimensions

- Cursor state discipline: `NSCursor` hide/unhide and push/pop fully balanced; `resetCursorState` self-healing at session entry; coordinators call `beginCrosshairSession` before `makeKeyAndOrderFront` (conventions satisfied).
- Flag-leak hygiene: `inFlightValidation`, `isShowingAlert`, capture flags all defer-cleared — the one exception is the Beautify `isProcessing` leak above.
- No `try!`/`as!` in production; no telemetry/analytics network calls; no `UNUserNotificationCenter` usage; history encryption (`HistoryCrypto`) used consistently for `history.enc`.
- Sparkle error paths and persistence errors are logged, not swallowed.
